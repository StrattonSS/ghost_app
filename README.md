G.H.O.S.T. – Ghost Hunting Field Journal
G.H.O.S.T. is a Flutter + Firebase app that turns your phone into a paranormal field notebook. It lets you explore haunted locations, log findings, track your investigation stats on a live leaderboard, and use device sensors (like the magnetometer) as lightweight investigation tools.

Features
Auth & profiles

Email/password registration and login with Firebase Authentication.

Per-user profile stored in Firestore, including username and optional date of birth.

Display name synced to Firebase Auth for consistent user identity.

Haunted locations

Browse and view haunted locations with name, city, state, coordinates, and metadata.

Dedicated detail screen that can deep-link via a locationId route argument.

Findings journal

Log field findings with:

Location name, city, state.

Evidence type (e.g., Observation, Magnetic Spike, Audio, Visual, Environmental Change).

Optional magnetic field reading.

Free-form notes.

Validated forms that return structured data back to the caller, which is then saved to Firestore.

Leaderboard

Firestore-backed leaderboard that tracks per-user:

locationsVisited count.

entriesLogged (evidence logged) count.

Real-time top list ordered by chosen metric (visited locations or evidence logged).

Banner showing current user’s rank and counts, with clear signed-in/signed-out messaging.

Automatic creation of a leaderboard document when an authenticated user is detected.

Tools / scanner system

Tools hub screen (“Scanner”) listing investigation tools.

Magnetic Field Reader page that uses the device magnetometer to show live field changes.

Clear warning that the EMF/magnetic tool is a field-use sensor only, not a calibrated scientific instrument.

Tool unlock & rewards

Local progression system using shared_preferences:

Ghost coins balance.

Per-tool unlock state (e.g., EMF Reader, UV Light, Spirit Box, Parabolic Mic, Camera).

Evidence logging can increment local counters and mark coins as claimable.

Unlocks and coin counts persist across app restarts.

Admin & configuration

Admin login helper that:

Signs in a user via email/password.

Checks a Firestore role field for admin.

Signs out immediately if the user is not an admin.

Designed to be paired with Firestore Security Rules for real role-based access control.

Navigation & structure

Bottom navigation with four tabs:

Locations (Home).

Findings (Journal).

Scanner (Tools).

Profile.

IndexedStack for tab content so each tab preserves state when you switch.

Video-based splash screen that plays once and then routes into the auth flow.

Tech stack
Framework: Flutter (Material, dark-themed UI).

Backend: Firebase (Auth, Firestore, Storage-ready).

Config: firebase_options.dart generated by FlutterFire CLI (DefaultFirebaseOptions.currentPlatform).

Packages (core concepts):

firebase_core – Firebase initialization.

firebase_auth – Email/password auth.

cloud_firestore – User profiles, leaderboard, journal data.

shared_preferences – Local unlocks and coins.

permission_handler – Camera, microphone, and related permissions.

video_player – Animated splash video.

Device sensor package(s) for reading magnetometer (EMF-style) data.
